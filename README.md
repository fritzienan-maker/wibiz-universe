# WiBiz Academy

> Phase 1 — GHL-connected certification portal

WiBiz Academy (`academy.wibiz.ai`) is the learning and certification portal for WiBiz clients. It sits on top of GoHighLevel (GHL), which is the source of truth for business identity, payment status, plan tier, and vertical.

**Phase 1 scope:** A paid GHL contact can be detected via webhook, provisioned with a portal account, log in with their temporary password, and access a placeholder dashboard while the full curriculum is built.

## Quick Start (Local Dev)

```bash
# 1. Clone the repo
git clone https://github.com/aileenwebdev/wibiz-academy.git
cd wibiz-academy

# 2. Install dependencies
pnpm install

# 3. Set up environment variables
cp .env.example .env
# Edit .env — fill in DATABASE_URL and JWT_SECRET at minimum

# 4. Run database migrations (creates all tables)
npx tsx scripts/migrate.ts

# 5. Seed the admin user (run once)
ADMIN_EMAIL=you@example.com ADMIN_PASSWORD=YourStr0ngPass npx tsx scripts/seed-admin.ts

# 6. Start the dev server
pnpm dev
# → http://localhost:3000
```

The dev server runs a single Express process that serves both the API and the Vite frontend with hot-module replacement.

## Environment Variables

Copy `.env.example` to `.env` and fill in all required values.

| Variable | Required | Description |
|---|---|---|
| `NODE_ENV` | Yes | `development` or `production` |
| `PORT` | No | Server port (default: `3000`) |
| `APP_BASE_URL` | Yes | Full URL e.g. `https://academy.wibiz.ai` |
| `DATABASE_URL` | Yes | PostgreSQL connection string |
| `JWT_SECRET` | Yes | Min 32 chars — signs session tokens |
| `JWT_EXPIRES_IN` | No | Token lifetime (default: `7d`) |
| `ADMIN_EMAIL` | Seed only | Email for the wibiz_admin account |
| `ADMIN_PASSWORD` | Seed only | Password for the wibiz_admin account |
| `GHL_API_KEY` | Yes | GHL Private Integration API key |
| `GHL_LOCATION_ID` | Yes | GHL sub-account location ID |
| `GHL_WEBHOOK_SECRET` | Yes | Shared secret — must match GHL workflow header |

> **Security:** Never commit `.env`. The `.gitignore` already excludes it.

## API Reference

### Auth

| Method | Path | Auth | Description |
|---|---|---|---|
| `POST` | `/api/auth/login` | — | Email + password → sets `wibiz_session` cookie |
| `POST` | `/api/auth/logout` | — | Clears `wibiz_session` cookie |
| `GET`  | `/api/auth/me` | ✓ | Returns current user profile |

### Dashboard

| Method | Path | Auth | Description |
|---|---|---|---|

### Admin (wibiz_admin only)

| Method | Path | Auth | Description |
|---|---|---|---|

| `GET` | `/api/admin/sync-events` | ✓ admin | Recent sync event log |

## Architecture Overview

```
GHL Contact
    │  Payment received → moves to "Onboarding Started" stage
    │  GHL workflow generates temporary_pass
    │
    ▼
POST /api/webhooks/ghl/provision
    1. Log raw payload → webhook_log
    2. Validate x-wibiz-secret header (401 if wrong)
    3. Validate required fields with Zod (400 if missing temporary_pass)
    4. Check ghl_contact_id in users table (200 if duplicate — idempotent)
    5. bcrypt.hash(temporary_pass, 12)
    6. INSERT INTO users (role: client_admin, activated_at: now)
    7. INSERT INTO sync_events (event_type: user_created)
    8. Return 200

User logs in at /login
    ↓
POST /api/auth/login
    → bcrypt.compare(password, hash)
    → jwt.sign({userId, email, role})
    → res.cookie("wibiz_session", token, { httpOnly, sameSite: none, secure })
    → 200 { user }

Protected routes
    → requireAuth middleware reads cookie
    → jwt.verify → attaches req.user
    → 401 if missing/expired
```

**Auth:** JWT stored in an `httpOnly` cookie (`wibiz_session`). The frontend never reads or stores the token — `credentials: 'include'` sends it automatically.

**Roles:**
| Role | Access |
|---|---|
| `client_admin` | Login, dashboard, change-password |
| `wibiz_admin` | All of the above + `/admin` routes |
| `client_staff`, `operator` | Phase 2+ |

## GHL Integration

### Webhook payload (what GHL must send)

```json
{
  "contact_id":    "GHL_CONTACT_ID",
  "email":         "user@example.com",
  "first_name":    "Jane",
  "last_name":     "Doe",
  "plan_tier":     "standard",
  "vertical":      "dental",
  "hskd_required": false,
  "temporary_pass": "TempPass123!",
  "location_id":   "your-ghl-location-id"
}
```

### Required header

```
x-wibiz-secret: <value of GHL_WEBHOOK_SECRET env var>
```

### GHL workflow setup

In the **"Wibiz Launch (SG) — Successful Payment"** workflow:

1. After contact reaches **Onboarding Started** stage
2. Add a **Send Webhook** action
3. URL: `https://academy.wibiz.ai/api/webhooks/ghl/provision`
4. Header: `x-wibiz-secret` = `<your GHL_WEBHOOK_SECRET>`
5. Map fields: `contact.id` → `contact_id`, `contact.email`, `contact.first_name`, `contact.last_name`, custom fields for `plan_tier`, `vertical`, `hskd_required`, `temporary_pass` (from `contact.temporary_pass` field), `location.id`
